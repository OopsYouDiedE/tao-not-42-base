# Kubric MOVi-E 数据集详细阐述 (TAO-NOT-42-Base)

本项目基于 **Kubric MOVi-E** 数据集，通过流式架构提取多维度物理量，用于训练具备物理直觉的视觉神经网络。以下是关于该数据集的加载机制、包含内容及各物理量的详尽阐述。

## 1. 数据集简介与获取方式
- **名称**: Kubric MOVi-E (`movi_e`)
- **来源**: 托管在 Google Cloud Storage 的 TensorFlow Datasets (TFDS) 库 (`gs://kubric-public/tfds`)
- **使用方式**: 
  - 训练代码无需手动提前下载巨大的完整数据集。`load_dataset.py` 利用 `tfds.load()` 自动进行流式加载 (Streaming)。
  - 为了最大化 I/O 效率，配置了 `interleave_cycle_length=16`，支持并行从 16 个文件中交叉读取数据。

## 2. 异步数据流管线 (Data Pipeline)
系统采用了高吞吐量的双缓冲异步架构，避免了 GPU 等待 CPU 处理数据（拒绝空转）：
1. **CPU 异步缓冲池 (`AsyncDataBuffer`)**: 
   - 运行在一个独立守护线程中，后台持续从网络/本地拉取视频片段，将其转换为 Tensor，存入 `deque` 环形缓冲池（最大容量64）。
   - 训练时按 `batch_size` 随机在缓冲池中抽样（无序混合），增加了训练的多样性。
2. **GPU 并发预处理器 (`CUDAPrefetcher`)**:
   - 使用独立的 CUDA Stream (`torch.cuda.Stream`)。
   - 在后台悄悄将 CPU 传来的 Batch 转移到 GPU 并执行极耗时的图像插值、Mask 构建和归一化处理。
   - 训练主循环只需调用 `prefetcher.next()` 即可瞬间拿到在 GPU 准备就绪的数据。

## 3. 包含的核心物理量与参数详解

送入网络的每一帧数据都经过了 `process_batch_on_gpu` 严格的标准化。以下是训练中核心物理量的上下限和实际含义。

### 3.1 视频数据 (Video)
- **原始数据**: uint8 RGB 图像。
- **处理方式**: 转换为 `float32` 并除以 `255.0`，同时通过双线性插值缩放到 `target_size` (默认 256x256)。
- **上下限**: `[0.0, 1.0]`
- **实际含义**: 供模型提取语义与物理特征的输入。

### 3.2 绝对深度 (Depth)
- **原始数据**: uint16，单位通常代表毫米 (mm)。
- **处理方式**: 
  - 除以 `1000.0` 转换为 **米 (meters)**。
  - **天空/背景处理**: 将深度等于 0 的区域判定为天空掩码 (`sky_mask_raw`)，并强行将其深度设置为 `100.0` 米，防止除零错误且符合“极其深远”的物理常识。
  - **截断保护**: 全局执行 `torch.clamp(depth, 0.01, 100.0)`。
- **上下限**: `[0.01, 100.0]` (米)
- **实际含义**: 像素点在三维空间中距离相机的真实物理距离。网络实际预测的是它的对数形式 `log_depth`，其区间为 `[-4.6, 4.6]`。

### 3.3 像素光流 (Forward Flow)
- **原始数据**: 储存在 uint16 中的压缩偏移量。
- **解码方式**: 依赖数据集提供的元数据 `[minv, maxv]` 进行线性拉伸：`flow / 65535.0 * (maxv - minv) + minv`。若元数据丢失则使用后备公式 `(flow - 32768.0) / 64.0`。
- **GPU 归一化**: 执行 `torch.clamp(flow_raw * 2.0 / target_size, -1.5, 1.5)`，将绝对像素偏移映射为相对于图片尺寸的比例。
- **上下限**: 被严格截断在 `[-1.5, 1.5]` 之间（即一次移动不得超过全图尺寸的 1.5 倍）。
- **实际含义**: t 时刻的像素点在 t+1 时刻的位置偏移向量。这约束了物理世界的局部一致性。

### 3.4 相机参数与自运动 (Camera / Ego Pose)
- **相机位置 (`cam_pos`)**: `[Batch, Time, 3]`。相机在世界坐标系下的 (X, Y, Z) 绝对坐标。
- **相机姿态 (`cam_quat`)**: `[Batch, Time, 4]`。相机在世界坐标系下的旋转四元数 (Quaternions)。
- **相对运动提取 (Ego Pose)**: 
  - 模型并非直接学习绝对坐标，而是在 `utils.py:compute_physics_loss` 中，通过前后两帧矩阵相乘，计算出相机的**相对平移 (T_delta)** 和**相对旋转矩阵 (R_delta)**。
  - R_delta 被展开成 6D 连续表征（利于神经网络回归），加上 3D 平移，形成最终预测的 **9D 位姿向量**。
- **相机内参 (Intrinsics)**:
  - 焦距通过视场角计算得到固定比例：`fx = fy = 35.0 / 32.0 * W`（例如当 W=256 时，`f = 280.0`）。
  - 光心为图像正中央：`cx = W/2, cy = H/2`。
  - 这套内参在计算重投影损失 (Photo Loss) 以及逆向 Warp (Inverse Warp) 时扮演了将深度+2D图像映射为3D点云的核心角色。

### 3.5 实例属性与动静解耦 (Instances & Dynamics)
- **分割掩码 (`segmentation`)**: 记录了画面中每一个孤立物体的唯一 ID。在预处理中，代码会根据 Mask 自动生成 Dense Bounding Boxes (`bboxes_dense`)，供 YOLO 头部学习目标检测。
- **运动属性 (`is_dynamic`)**:
  - 静态背景 / 静态刚体 (False) 映射为分类目标：`Class 0`。
  - 独立的动态物体 (True) 映射为分类目标：`Class 1`。
  - **实际含义**: 迫使模型在检测物体的同时，学会物理层面的**动静解耦**。即分辨哪些物体的像素移动是因为“相机自身移动（Ego Motion）”引起的（静态背景视差），哪些是因为“物体自己长了腿/受到外力”移动引起的（独立运动）。
