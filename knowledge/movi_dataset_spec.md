# Kubric MOVi 数据集家族详细阐述 (TAO-NOT-42-Base)

本项目基于 Google 发布的 **Kubric MOVi (Multi-Object Video)** 合成数据集家族。该数据集系列专为物理直觉学习、目标发现、分割与跟踪设计，通过逐步增加视觉与物理动态的复杂度，全面考验模型的理解能力。

## 1. MOVi 家族演进史 (MOVi-A 到 MOVi-F)

Kubric 团队将该系列分为了 A 到 F 六个难度递增的版本。当前代码 (`load_dataset.py`) 中默认使用的是 `tfds.load("movi_e")`，但您可以根据训练需求无缝切换到其他变体。以下是全家族的复杂度差异：

| 变体名称 | 场景复杂度与特征 | 核心挑战与适用任务 |
| :--- | :--- | :--- |
| **MOVi-A** | **最基础:** 类似 CLEVR 数据集。灰色地板，固定光源，包含 3–10 个简单的几何体（纯色立方体、球体等）。相机固定，仅有轻微随机抖动。 | 基础对象发现、简单的刚体物理。 |
| **MOVi-B** | **纹理升级:** 场景依然简单，但在 MOVi-A 的基础上增加了更多复杂的几何形状，并为所有物体表面引入了高频的随机纹理。 | 基础纹理抗干扰、边界感知。 |
| **MOVi-C** | **真实世界视觉:** 质的飞跃。简单几何体被替换为来自 Google 扫描对象 (GSO) 的**真实 3D 物体**。背景和光照被替换为真实的 HDRI 全景穹顶。 | 零样本真实世界泛化、复杂光影渲染。 |
| **MOVi-D** | **动静结合:** 增加物体总数量至 10–20 个。绝大多数物体作为障碍物静止铺在地面上，仅有 1–3 个物体被**抛入场景中**进行动态运动。 | 动静解耦、杂乱背景下的运动跟踪。 |
| **MOVi-E** | **相机移动 (当前默认):** 继承 D 的所有复杂度，并且引入了**线性的摄像机平移运动**。相机在场景半球壳上以恒定速度滑轨移动。 | 摄像机自运动 (Ego-motion) 与物体独立运动的解耦、深度估计。 |
| **MOVi-F** | **光流与运动模糊:** 与 E 场景一致，但默认渲染分辨率提升至 512x512，且在视频帧中强制引入了**随机运动模糊 (Motion Blur)**。 | 极限光流预测、抗运动模糊的特征感知。 |

*注: MOVi C、D、E 包含专门的 "test" 拆分集，其中使用了训练集中未出现过的 GSO 物体和 HDRI 背景，专门用于评估模型在未知环境下的泛化能力。*

## 2. 数据集获取方式与异步管线
- **来源**: 托管在 Google Cloud Storage 的 TFDS 库 (`gs://kubric-public/tfds`)。支持按分辨率拉取 (如 `movi_e/256x256`)。
- **流式管线 (Data Pipeline)**:
  - `AsyncDataBuffer`: 系统的 CPU 守护线程异步从 TFDS 流式读取数据，转换为 PyTorch 张量并放入 `deque` 环形缓冲池，防止网络卡顿拖慢训练。
  - `CUDAPrefetcher`: 后台通过专属 CUDA Stream 并发将数据搬运至 GPU，提前完成高负载的尺寸缩放、掩码预计算和归一化处理。主训练循环无需等待。

## 3. 核心物理量参数详解 (当前模型已接入)

送入网络的每一帧数据都经过了 `process_batch_on_gpu` 的严格标准化处理。以下是代码中各类物理量的尺寸、边界限制及其真实含义：

### 3.1 视频数据 (Video)
- **原始数据**: uint8 格式的 RGB 视频帧。
- **预处理方式**: 转换为 `float32` 并除以 `255.0`，通过双线性插值缩放到目标网络分辨率 `target_size` (默认 256x256)。
- **上下限**: `[0.0, 1.0]`

### 3.2 绝对深度 (Depth)
- **原始数据**: uint16，原始单位为毫米 (mm)。
- **预处理方式**: 
  - 缩放: 除以 `1000.0` 转换为 **米 (meters)**。
  - 天空/无限远处理: 提取 `depth == 0` 的区域作为天空掩码，并将这些区域的深度强行设为 `100.0` 米，防止对数运算错误并符合物理直觉。
  - 截断保护: 全局执行 `torch.clamp(depth, 0.01, 100.0)`。
- **上下限**: `[0.01, 100.0]` (米)
- **模型预测**: 网络并不会直接输出线性深度，而是预测其对数形式 `log_depth`，其理论值域被映射到了 `[-4.6, 4.6]`。

### 3.3 稠密光流 (Forward Flow)
- **原始数据**: uint16 压缩偏移量。
- **解码公式**: 根据元数据提供的 `[minv, maxv]` 进行精准恢复：`flow / 65535.0 * (maxv - minv) + minv`。若无元数据则使用后备缩放 `(flow - 32768.0) / 64.0`。
- **GPU 归一化**: `torch.clamp(flow_raw * 2.0 / target_size, -1.5, 1.5)`。它将绝对像素偏移映射为相对于图片尺寸的比例。
- **上下限**: 严格截断在 `[-1.5, 1.5]` 之间（限制单步最大移动不得超过画面尺寸的 1.5 倍，过滤掉瞬间闪烁或爆炸性渲染错误）。
- **实际含义**: 描述 t 时刻像素点在 t+1 时刻的位置二维移动向量，是引导网络学习时间连续性的核心。

### 3.4 相机运动轨迹 (Camera & Ego Pose)
- **相机位置 (`cam_pos`)**: `[Batch, Time, 3]`。描述相机在 3D 世界坐标系下的 (X, Y, Z) 绝对坐标。
- **相机姿态 (`cam_quat`)**: `[Batch, Time, 4]`。描述相机的绝对旋转四元数 (Quaternions)。
- **相对位姿推算 (Ego Motion)**: 
  - 模型在训练时不预测绝对坐标。`utils.py:compute_physics_loss` 中，系统会通过逆矩阵乘法，求出前后两帧相机的**相对平移 (3D)** 和**相对旋转矩阵**。
  - 旋转矩阵被展平为连续的 6D 表征，配合 3D 平移，合并为供模型学习的 **9D 相对位姿向量**。
- **相机内参 (Intrinsics)**:
  - 焦距推导: `fx = fy = 35.0 / 32.0 * W`（当 W=256 时为 280.0）。
  - 主点位置: `cx = W/2, cy = H/2`。
  - 该内参与 Kubric 引擎视角 (FOV) 完美对齐，是重投影损失 (Photo Loss/Warp) 能够成功运行的数学基础。

### 3.5 实例属性与动静解耦 (Instances & Dynamics)
- **实例分割 (`segmentation`)**: 包含每个对象的唯一 ID。系统依据此掩码，自动框选出密集目标边界框 (`bboxes_dense`)，用以监督网络的目标检测层。
- **运动属性 (`is_dynamic`)**:
  - 静止物体 / 背景地面 (`False`) 映射为 `Class 0`。
  - 空中飞行 / 受到碰撞移动的物体 (`True`) 映射为 `Class 1`。
  - **核心目的**: 在 MOVi-E/F 等高级别数据集中，画面中的像素都在移动。这个标签强迫模型分清：哪些视差是因为**摄像机自己在移动**（Ego-motion 导致的背景退行），哪些是因为**物体不受摄像机影响自己在飞**（独立物理运动）。这是使得网络拥有人类物理直觉的最重要机制。

## 4. 尚未接入的富物理属性 (Hidden Native Capabilities)

虽然当前项目的 `load_dataset.py` 仅解析了上述基础物理量，但 Kubric MOVi 数据集的原始 TFDS 记录中**原生包含了极其丰富的底层物理参数**（由 PyBullet 物理引擎直接导出）。如果您未来想要进一步增强模型的“物理学直觉”，随时可以从数据集中解析出以下字段：

### 4.1 物体动力学参数 (Object Dynamics)
- **速度 (`instances.velocities`)**: `[Time, Num_Instances, 3]`。记录了每个物体在每一帧的三维线速度向量 (X, Y, Z)。
- **角速度 (`instances.angular_velocities`)**: `[Time, Num_Instances, 3]`。记录了每个物体绕自身的旋转角速度。
- **3D位姿 (`instances.positions` / `quaternions`)**: 每个物体在世界坐标系中的绝对坐标和旋转四元数。
- *潜在用法*: 可以用来直接监督网络输出未来的速度向量，实现显式的物理外推引擎。

### 4.2 物体物理材质属性 (Material Properties)
- **质量 (`instances.mass`)**: `[Num_Instances]`。物体的绝对重量。
- **摩擦系数 (`instances.friction`)**: `[Num_Instances]`。表面粗糙度。
- **弹力/恢复系数 (`instances.restitution`)**: `[Num_Instances]`。物体碰撞后的弹跳系数 (Bounciness)。
- *潜在用法*: 用于训练系统去通过视觉“估测”物体的重量和材质，预测两次弹跳之间的高度衰减。

### 4.3 高阶相机与图像参数 (Advanced Camera/Image Specs)
- **焦距标量 (`camera.random_focal_length`)**: 真实的渲染焦距。
- **法线图 (`normal`)**: `[Time, Height, Width, 3]`。记录了图像中每个像素对应 3D 表面法线的 (X, Y, Z) 向量。
- **3D 边界框 (`instances.bboxes_3d`)**: 每个实例极其精确的 3D 立方体边界框坐标。
- **反向光流 (`backward_flow`)**: 补充的 `t` 到 `t-1` 帧的光流，对于训练双向时序一致性（Forward-Backward Consistency）非常有效。
