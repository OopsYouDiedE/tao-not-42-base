# FeaturePredictorHead (时空特征动力学预测器)

`FeaturePredictorHead` 是自监督物理模型中用于发掘“非常规独立运动物体（物理异常检测）”的核心模块。它充当了一个“微型物理引擎”，基于历史特征与相机的已知运动（动作），预测未来时刻在同一个物理坐标上应当呈现的场景特征。

---

## 1. 设计初衷与位置

在时序追踪中，为了让网络拥有“物理常识（Physics Common Sense）”并能自主发现哪些物体正在独立移动，我们需要一个无需标注的度量形式。

`FeaturePredictorHead` 实现了一种优雅的**自监督特征预测差值检测机制**：
- **物理预测**：对于被判定为静态背景/刚体的特征格点，其未来的特征演化应当是“相机运动带来的视差改变”。
- **异常捕获**：如果我们利用 $t$ 时刻的特征 $F_t$ 和相机的相对位姿 $A_{t \to t+1}$，去预测 $t+1$ 时刻的特征 $\hat{F}_{t+1}$，则对于完全静止的物体，$\hat{F}_{t+1}$ 应该极其接近真实值 $F_{t+1}$；而对于**正在独立加速、下落或漂移的物体**，因为其运动不符合相机的自运动规律，预测特征与真实特征之间会产生极大的 L1 偏差。
- 这个偏差图（`anomaly_map`）被用于自动圈定动态物体，驱动网络进行重点追踪。

---

## 2. 类接口与参数说明

### 构造函数

```python
def __init__(self, channels=256, action_dim=9):
```

| 参数 | 类型 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- |
| `channels` | `int` | `256` | 时空输入特征的通道维度（通常对应 Mamba 混合特征的 128 或 256 维）。 |
| `action_dim` | `int` | `9` | 相机 Ego-Pose 动作向量维度。 |

---

## 3. 前向传播与特征拼接机制

`FeaturePredictorHead` 采用多层 Bottleneck 架构来提取复杂的动力学相关性：

```python
def forward(self, state, action):
```

1. **动作广播拼接 (Action Broadcasting)**：
   相对位姿 `action`（维度为 `[B, 9]`）首先在空间轴上进行大范围广播，形成形状为 `[B, 9, H, W]` 的动作空间矩阵。接着将其与输入的历史时空特征 `state`（形状为 `[B, C, H, W]`）进行通道融合（`torch.cat`），形成 $C + 9$ 维的混合输入张量。
2. **深度 Bottleneck 提取**：
   - 首先通过 $1 \times 1$ 的首部卷积层 `self.stem` 将通道数重新整理回 `channels` 维，以保证网络的计算效率。
   - 随后送入由多级 `Bottleneck` 组成的动力学残差块进行物理空间重映射。
3. **输出预测**：
   最后一层经过 $3 \times 3$ 特征卷积输出预测的特征图 $\hat{F}_{t+1}$（与输入的 `state` 形状完全相同），并送入主训练损失引擎进行 Smooth L1 差值计算，自动计算并输出自监督物理异常损失 `loss_anom`。
