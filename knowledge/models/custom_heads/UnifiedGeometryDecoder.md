# UnifiedGeometryDecoder (三维物理几何解码器)

`UnifiedGeometryDecoder` 是模型用来对场景三维几何（绝对深度）与物理运动（稠密前后向光流）进行重建和预测的“渲染引擎”。它接收骨干与时空 Mamba 混合的多尺度表征，并通过上采样和连续跳跃连接恢复至物理像素坐标。

---

## 1. 设计初衷与位置

自主移动的相机视角（Ego-motion）使得直接预测运动物体的物理流极具挑战。为了彻底解耦自主相机视差带来的伪物理偏移，`UnifiedGeometryDecoder` 巧妙地引入了**相机自运动相对位姿的特征耦合机制**。

它在系统中所处的位置如下：
- 接收 FPN 输出的 `f1`、`f2`（浅层时序卷积特征）以及时空混合后的 `p3` 核心特征图。
- 接收 `EgoPoseHead` 估计出的 9D 相机相对位姿参数。
- 并行解码输出 $T$ 长度的 **绝对深度图**（Depth Map）与 **对齐光流图**（Optical Flow Map），用于驱动后端的自监督光度重建误差约束。

---

## 2. 类接口与参数说明

### 构造函数

```python
def __init__(self, ch_p3=256, ch_f2=96, ch_f1=48, pose_dim=9):
```

| 参数 | 类型 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- |
| `ch_p3` | `int` | `256` | 时空 P3 特征图的输入通道数。 |
| `ch_f2` | `int` | `96` | 浅层特征 $F_2$ 的通道数。 |
| `ch_f1` | `int` | `48` | 极浅特征 $F_1$ 的通道数（用以维持超高边缘保真度）。 |
| `pose_dim` | `int` | `9` | 相机 Ego-Pose 连续 6D+3D 的位姿参数维度。 |

## 3. 前向传播计算拓扑

`UnifiedGeometryDecoder` 的数据流图设计采用类似 U-Net 的多级上采样金字塔拼接结构：

```
P3 特征图  --------> [Upsample2x + Conv] --+
                                         |-> [Concat] --> [Conv1] ---------> [Upsample2x + Conv] --+
F2 浅层特征 -----------------------------+                                                         |-> [Concat] --> [Conv2]
                                                                                                    |                        |
F1 极浅特征 -----------------------------------------------------------------------------------------+                        v
                                                                                                                   [分支解码]
                                                                                                                  /          \
                                                                                                                 /            \
                                                                                                      [Depth Branch]         [Flow Branch]
                                                                                                            |                      |
                                                                                                       绝对深度图 (1ch)      稠密光流 (2ch)
                                                                                                                          (融入 Ego-Pose 特征)
```

### 3.1 绝对深度估计分支 (Depth Estimation Branch)
1. 金字塔解码器融合高分辨率浅层边缘特征 `f1` 和 `f2`。
2. 深度估计分支通过对经过二次上采样的稠密特征进行卷积投影，输出 1 个通道的对数深度图。
3. 对数深度在顶层 `TAONot42VisionModel` 中被自动还原为物理绝对深度：
   $$\text{depth} = \exp(\text{clamp}(\text{depth\_raw}, \min=-4.6, \max=4.6))$$
   极大地保证了其数值不会发散，同时提供了对远景天空物体的稳定边界刻画。

### 3.2 Ego-motion 引导的光流分支 (Ego-motion Guided Flow Branch)
1. 稠密运动光流受相对位姿 `ego_pose` 强力指引。
2. 如果满足需要预测光流的条件（当前训练阶段激活）：
   - 相机相对位姿 $9\text{D}$ 向量在空间维度进行广播扩展，形成 $H_{\text{flow}} \times W_{\text{flow}} \times 9$ 的位姿引导特征图。
   - 该位姿特征图与光流浅层特征图拼接（`torch.cat`），并送入 `flow_conv`，这让光流预测能直接学习到“由相机旋转平移带来的视差偏移分量”，从而物理式地滤除虚假背景光流，只专注于动能独立物体的精确运动速度估计。
