# Proto26 (多尺度实例分割原型生成模块)

`Proto26` 是 YOLOE 架构中负责生成高分辨率实例分割原型（Instance Prototypes）的子模块。它通过聚合浅层和深层的多尺度金字塔特征，并行输出语义分割背景掩膜与 32 通道的实例分割原型图。

---

## 1. 设计初衷与解耦机制

在基于系数法（Coefficients-based）的实例分割算法中（如 YOLACT 和 YOLOE-Seg），实例的最终分割掩膜（Mask）并不是通过卷积直接暴力预测出来的，而是通过**爱因斯坦求和矩阵运算（Einsum）**合成：
- **语义基础（Prototypes）**：由 `Proto26` 首先融合多尺度信息，并上采样输出一个高分辨率、32 通道的“掩膜积木基础图库” `proto`（形状为 `[B, 32, H_proto, W_proto]`，通道数 $32$ 代表 32 种不同的基向量，通常分辨率为原图的 $1/4$，即 $160 \times 160$）。
- **个性权重（Coefficients）**：由 `YOLOESegment26` 的各尺度 One-to-One 预测层，为每个检测出的实例预测一个 32 维的个性化投影系数向量 $\mathbf{C} \in \mathbb{R}^{32}$。
- **矩阵合成**：对于某一个存活的实例，其最终的实值分割掩膜直接由个性系数与基础图库相乘求和得出：
  $$\text{Mask}_{\text{raw}} = \sum_{k=1}^{32} C_k \times \text{proto}_k$$
- **优势**：将密集的像素级回归解耦为“1个共享的高分辨率特征库+少量的低维系数线性乘加”，使得网络能在保持超快推理速度的同时，获得极为锋利、高清的实例分割边缘线。

## 2. 类接口与参数说明

### 构造函数

```python
def __init__(self, ch, npr=256, nm=32, nc=80, semseg_nc=80):
```

| 参数 | 类型 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- |
| `ch` | `list` | - | 跨尺度特征输入通道列表（通常为 `[128, 256, 512]`）。 |
| `npr` | `int` | `256` | 原型图解码器的内部核心隐空间维度。 |
| `nm` | `int` | `32` | 实例原型特征图的输出通道数（当前为 32）。 |
| `semseg_nc` | `int` | `80` | 官方预训练权重中，辅助语义分割（`semseg`）的通道数（通常设为 80，与 COCO 类数对齐）。 |

## 3. 特征聚合与语义解耦计算

### 3.1 多级特征融合 (Multi-scale Fusion)
`Proto26` 对多尺度特征 [P3, P4, P5] 进行上采样并逐层累加融合，确保网络既捕获局部浅层的物体纹理，也囊括深层低分辨率特征的物体全局结构：
```python
feat = x[0]  # P3 特征图
for i, f in enumerate(self.feat_refine):
    # P4, P5 特征通过 1x1 降维并经过双线性/最近邻插值累加到 P3 上
    feat = feat + F.interpolate(f(x[i+1]), size=feat.shape[2:], mode="nearest")
```

### 3.2 独立语义分割解耦 (semseg)
为了让实例原型 `proto` 的表示更加纯净，`Proto26` 独立设计了辅助的多级深度卷积 `self.semseg`：
- 在融合特征图上并行预测一个 80 维的稠密语义分割图 `semseg`。
- **权重对齐保护**：由于官方权重在导出时在 `semseg` 分支中保留了 COCO-80 维的经典分类空间，而我们自监督追踪使用的是大词表 nc=4585。通过将 `semseg` 分支的通道解耦为 `semseg_nc=80`，可以安全地加载官方分割参数，绝对防范由于分类维数变化导致的主干权重初始化崩盘。
